# 通道消息

## 一、功能概述

- 一句话定义：
通道用户“@提及”是数字员工会话输入框中的对象选择能力，用于在消息输入过程中选择数字员工及通道用户，并以标签形式插入输入框。

- 功能目的：
解决用户在数字员工对话中需要快速指定协作对象的问题，实现数字员工与通道用户的统一提及入口、统一筛选逻辑与统一展示规范，降低选择成本和识别歧义。

- 目标用户：
普通业务用户。

## 二、典型使用场景

在数字员工会话中，通过“@提及”数字员工或通道用户，补充消息协作对象

- 触发条件：
用户在数字员工会话页中输入消息时，需要指定一个数字员工或一个/多个通道用户作为当前消息上下文中的协作对象，因此在输入框中点击 @ 按钮或直接输入“@”。

- 问题原因：
当前会话对象中既存在数字员工，也存在来自不同通道的联系人。如果缺少统一的选择入口与筛选规则，用户无法快速定位目标对象，且容易混淆数字员工与通道用户的语义差异。

- 用户目标：
用户通过“@提及”能力，快速找到并选择目标数字员工或通道用户，使消息输入过程更高效，且所选对象在输入框中可被清晰识别。

## 三、关键概念（本文提及）

|  概念名称  |  定义  |  作用  |  背景补充  |
|  --  |  --  |  --  |  --  |
| 数字员工 | 平台中可参与业务协作的智能体对象，如「库存优化助手」「计划协同助手」。 | 承接当前消息上下文中的智能处理能力，是用户会话的核心智能协作对象。 | 与通道用户同属于“@提及”候选对象，单次输入上下文仅允许生效1个。 |
| 通道用户 | 来自外部 IM 通道的联系人对象，如钉钉用户、飞书用户。 | 在当前消息中补充被提及的协作人员，支持多选。 | 按通道维度分组展示，多通道存在同名用户，需展示通道信息做区分。 |
| 提及下拉框 | 用户输入 @ 后弹出的候选对象选择面板。 | 承载数字员工与通道用户的分组展示、搜索过滤、查看更多/收起、空状态提示等交互能力。 | 面板宽度固定，高度在最小、最大值区间内自适应变化。 |
| 内联块 | 选择对象后插入到输入框中的标签化展示单元。 | 承载已选对象，与普通文本形成视觉明确区分。|数字员工、通道用户两类内联块，在样式、文案结构上需做明显差异化设计。 |

## 五、整体业务逻辑说明

这个流程是消息输入框中“@提及”功能的处理逻辑，分为「@通道用户」和「@数字员工」两条分支，核心规则是：

- 通道用户可以一次多选多个；
- 数字员工在单次输入中只能生效一次，且仅在会话页输入框中可用。

## 六、操作流程

### 操作名称：唤起“@提及”下拉框

入口：在数字员工会话页的消息「输入框」

操作步骤：
1. 点击输入框内 @ 按钮，或在输入框中输入 @。
2. 系统展示 “@提及”下拉框。
3. 系统根据当前输入框中是否已选择数字员工，决定候选对象范围。

结果：
- 当输入框中未存在数字员工内联块时，下拉框展示数字员工分组与通道用户分组。
- 当输入框中已存在数字员工内联块时，下拉框仅展示通道用户分组。

异常处理：
- 若当前候选对象数据为空，则展示空状态：无匹配的联系人。

### 操作名称：选择数字员工

入口：在“@提及“”下拉框中，点击我的数字员工分组下的某个数字员工。

操作步骤：

1. 在下拉框中查看数字员工列表。
2. 点击某个数字员工。
3. 系统在输入框中插入数字员工内联块。
4. 下拉框关闭，光标回到输入框中。

结果：
- 输入框中新增一个数字员工标签。
- 当前输入上下文状态变为“已选择数字员工”。
- 用户再次输入 @ 时，下拉框不再展示数字员工分组。

异常处理：
- 若数字员工插入失败，则保持下拉框不关闭，并提示操作失败原因（如有统一错误提示机制，则按系统规范执行）。


### 操作名称：选择通道用户

入口：在提及下拉框中，点击某个通道用户分组下的用户项。

操作步骤：

1. 在下拉框中查看通道用户分组。
2. 点击目标通道用户。
3. 系统在输入框中插入通道用户内联块。
4. 下拉框关闭，光标回到输入框中。

结果：
- 输入框中新增一个通道用户标签。
- 用户可继续输入消息内容，或继续输入 @ 选择其他通道用户。

异常处理：
- 若用户数据缺失或插入失败，则保持当前输入内容不变，并按系统统一错误提示方式反馈。

### 操作名称：搜索过滤提及对象

入口：在消息输入框中输入 @ 后，继续输入关键字。

操作步骤：

1. 输入 @ 后继续输入关键字。
2. 系统对候选对象进行实时过滤。
3. 系统刷新下拉框结果。

结果：
- 若存在匹配结果，则展示过滤后的数字员工与 / 或通道用户。
- 若当前已存在数字员工内联块，则仅展示匹配到的通道用户。
- 若无匹配结果，则展示 无匹配的联系人。

异常处理：
- 若搜索请求失败，则展示空态或统一异常提示，避免页面卡死或列表停留在旧状态。

### 操作名称：展开 / 收起通道用户列表

入口：在提及下拉框中，某通道分组命中用户数超过 5 条时，点击【查看更多】或【收起】。

操作步骤：

1. 当某通道命中结果超过 5 条时，系统默认展示前 5 条，并显示【查看更多】。
2. 点击【查看更多】。
3. 系统展开该通道全部命中结果，并将按钮文案切换为【收起】。
4. 点击【收起】。
5. 系统恢复默认 5 条展示状态。

结果：
- 展开后，当前通道显示全部候选用户。
- 收起后，当前通道恢复为默认 5 条展示。
- 其他通道分组状态不受影响。

异常处理：
- 若展开数据加载失败，则保持当前收起状态，并提示加载失败。

### 操作名称：删除已选数字员工后重新唤起提及

入口：在消息输入框中删除已存在的数字员工内联块后，再次点击 @ 按钮或输入 @。

操作步骤：

1. 删除输入框中的数字员工内联块。
2. 系统同步更新当前输入上下文状态。
3. 再次触发提及下拉框。

结果：
- 下拉框恢复展示数字员工分组与通道用户分组。

异常处理：
- 若状态未同步更新，则按输入框实际内容重新校验当前已选对象，确保数字员工分组显示逻辑正确。
